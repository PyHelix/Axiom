NEW EXPERIMENT: FEATURE COMPETITION DYNAMICS
=============================================
Script: feature_competition_dynamics.py
Rationale: Building on our lazy-vs-feature-learning findings, this experiment
studies gradient starvation — how easy-to-learn (high-SNR) features dominate
gradient flow and suppress learning of harder features during training.

Design:
  - Synthetic dataset with 3 feature groups at different signal-to-noise ratios:
    Strong (SNR=3.0, dims 0-4), Medium (SNR=1.0, dims 5-9), Weak (SNR=0.3, dims 10-19)
  - Binary classification, N=1000 train / N=300 test
  - Fully-connected networks: 3 hidden layers, ReLU + sigmoid output
  - Configuration sweep: widths [32, 64, 128, 256] x learning rates [0.01, 0.05, 0.1]
  - Measurements every 20 epochs (300 total):
    * Per-feature-group gradient magnitudes
    * Per-feature-group probing accuracy (linear probe on hidden activations)
    * Training/test accuracy and loss
    * Weight norms per layer

Scientific questions:
  1. Do easy features consistently suppress hard feature learning?
  2. Does wider networks mitigate or amplify gradient starvation?
  3. Is there a phase transition in feature learning order?
  4. How does this connect to our lazy-vs-feature learning findings?

Initial validation (local): Mean gradient ratio (strong/weak) = 1.39, with wider
networks showing stronger starvation effect (up to 1.84 for width=256). Strong
features achieve near-perfect probing accuracy from epoch 1; weak features lag.

Reference: Pezeshki et al. (2021), "Gradient Starvation: A Learning Proclivity
in Neural Networks" (inspired the investigation).
