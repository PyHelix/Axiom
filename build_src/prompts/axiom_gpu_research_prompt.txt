### MINIMUM GPU PROBLEM SIZES (MANDATORY -- enforced, not suggested)

**A GPU experiment that doesn't saturate the GPU is WORSE than a CPU experiment** -- it wastes a GPU slot that could run real GPU work while achieving CPU-like throughput. The following minimums are NON-NEGOTIABLE:

| Operation Type | Minimum Size | Why |
|---|---|---|
| Matrix eigendecomposition | **N >= 512** (512x512 matrix) | Below 512, CPU is faster than GPU after kernel launch overhead |
| Matrix multiply / SVD / linear solve | **N >= 1024** (1024x1024) | cuBLAS needs large matrices to saturate SMs |
| Monte Carlo / random sampling | **>= 100,000 samples per GPU call** | Batch all samples into one cupy operation, not a Python loop |
| Lattice / grid simulations | **>= 512x512 grid** (or 256x256x256 3D) | Small grids don't justify GPU memory transfer cost |
| Agent-based models | **>= 10,000 agents as GPU arrays** | Agents must be vectorized cupy arrays, not Python objects |
| Parameter sweeps | **>= 100 parameter combos batched as GPU tensor dims** | Don't loop in Python -- stack parameters as array dimensions |

**CuPy linalg limitations — functions that DO NOT EXIST in cupy:**
- cupy.linalg.eig() — does NOT exist. Use numpy.linalg.eig(cp.asnumpy(matrix)) instead (pull to CPU).
- cupy.linalg.eigvals() — does NOT exist. Use numpy.linalg.eigvals(cp.asnumpy(matrix)) instead.
- cupyx.scipy.linalg — does NOT reliably exist across CuPy versions. Do NOT import `cupyx.scipy.linalg`. Use `cupy.linalg` (eigh, eigvalsh, svd, solve) or fall back to `numpy.linalg`/`scipy.linalg` via `cp.asnumpy()`.
- CuPy ONLY has eigh() and eigvalsh() (symmetric/Hermitian matrices).
- For non-symmetric eigenproblems, you MUST fall back to numpy. This is fine for small matrices (< 512x512).
- For large non-symmetric matrices, use iterative methods (Arnoldi/power iteration) implemented in cupy directly.

**CuPy RandomState limitations:**
- cupy.random.RandomState does NOT have .random() method. Use rng.random_sample() instead (works on both numpy and cupy).
- For generating uniform random arrays, use rng.random_sample(shape) not rng.random(shape).

**Anti-patterns that WASTE GPUs (DO NOT DO THESE):**
- Calling `xp.linalg.eigvals()` on a 100x100 matrix inside a Python for-loop -- **this is CPU work pretending to be GPU work**
- Using cupy for array creation but numpy for the actual computation
- Running the same small operation 1000 times in a Python loop instead of batching into one large GPU operation
- Matrix sizes that scale with `species_count=96` or similar tiny numbers -- these belong on CPU
- Any experiment where the GPU utilization would average < 50%

**The test is simple: if your experiment runs equally fast on CPU and GPU, it should be a CPU experiment.** GPU experiments must demonstrate a >= 5x speedup over numpy to justify the GPU slot.

### PREFERRED PATTERN: Iterative Deepening Bisection (GPU)

**This is the PRIMARY pattern for GPU experiments.** Same Stockfish-style approach as CPU, but with cupy for O(N³) operations and GPU-scaled matrix sizes. GPUs can reach N=4096-8192 (vs CPU's 2048), producing much sharper results.

#### The iterative deepening pattern (GPU version):
1. Start at matrix size scaled from GPU FLOPS calibration (minimum N=512)
2. Do 8 binary search steps at that size (bisect the parameter)
3. Write result to disk (checkpoint — the answer is always ready)
4. Free GPU memory (`cp.get_default_memory_pool().free_all_blocks()`)
5. Carry forward ONLY the bracket (two floats: low, high)
6. Double N up to GPU memory limit
7. Bisect within the narrowed bracket from the previous pass
8. **TIME ESTIMATE: if `last_pass_time * 8 > time_remaining`, STOP**
9. Time runs out → last written result IS the answer

#### Why GPU iterative deepening is powerful:
- **GPU eigvalsh on N=4096 takes seconds** (vs minutes on CPU). More passes = tighter brackets.
- **GPU FLOPS calibration scales the starting N** — RTX 4090 starts at N=2048, GTX 1060 at N=800. Both stay busy for 30 min.
- **N cap scales with GPU VRAM** — For dense eigensolve (eigvalsh, eigvals): EIGH_VRAM_DIVISOR = 32768.0. Formula: MAX_N = int(sqrt(0.80 * GPU_MEMORY_MB * 1e6 / 32768.0)). Divisors below 200 are BANNED — they cause cudaErrorIllegalAddress on volunteer GPUs. 8GB GPU caps at ~443, 16GB at ~626, 24GB at ~767.
- Same anytime algorithm: result is always valid after each completed pass.
- Memory stays flat: one NxN matrix on GPU at a time, freed between passes.

#### What makes a GOOD GPU iterative deepening experiment:
- A **parameter with an unknown critical threshold** (phase transition, stability boundary, universality breakdown)
- An **observable** that cleanly distinguishes two phases (fraction, ratio, density, level spacing ratio)
- A **matrix operation that scales as O(N³)** — `cp.linalg.eigvalsh()`, `cp.linalg.eigvals()`, SVD
- A **genuinely OPEN problem** — no known analytical solution exists
- The **bisection parameter** should have a roughly constant critical value across N

```python
            eigs = cp.asnumpy(eigs)  # back to numpy for analysis
        total += your_criterion(eigs)
    # Free GPU memory between measurements
    if _HAS_GPU:
        cp.get_default_memory_pool().free_all_blocks()
    return total / n_samples


def save_result(p_low, p_high, current_N, passes_completed,
                total_eigdecomps, elapsed):
    """Write current best answer to disk (anytime algorithm)."""
    _bw = max(float(p_high - p_low), 1e-15)
    result = {
        "experiment_result": {

                "param_low": float(p_low),
                "param_high": float(p_high),
                "param_c_estimate": float(0.5 * (p_low + p_high)),
                "bracket_width": float(p_high - p_low),
                "passes_completed": float(passes_completed),
                "total_eigdecomps": float(total_eigdecomps),
                "gpu_tflops": float(GPU_TFLOPS),
            },
            "hypothesis": {
                "param_c_estimate": float(0.5 * (p_low + p_high)),
                "bracket_width": float(p_high - p_low),
                "deepest_N": float(current_N),
                "passes_completed": float(passes_completed),
            },
        }
    }
    with open("experiment_result.json", "w") as fh:

def main():
    t0 = time.time()
    deadline = t0 + _run_duration - 10

    param_low = PARAM_LOW_INIT
    param_high = PARAM_HIGH_INIT
    N = INITIAL_N
    passes_completed = 0
    total_eigdecomps = 0
    last_pass_time = None
    sample_seed = _seed

    while time.time() < deadline:
        # TIME ESTIMATION: don't start a pass you can't finish
        if last_pass_time is not None:
            if time.time() + last_pass_time * 8 > deadline:
                break

        pass_start = time.time()
        pass_low, pass_high = param_low, param_high
        samples = max(2, DISORDER_SAMPLES - passes_completed)

        for step in range(BISECTION_STEPS):
            if time.time() >= deadline:
                break
            mid = 0.5 * (pass_low + pass_high)
            obs = measure_observable(N, mid, samples, sample_seed)
            sample_seed += samples * 1000
            total_eigdecomps += samples

            if obs > OBSERVABLE_THRESHOLD:
                pass_low = mid
            else:
                pass_high = mid

        # Tighten bracket (NEVER widen)
        param_low = max(param_low, pass_low)
        param_high = min(param_high, pass_high)

        last_pass_time = time.time() - pass_start
        passes_completed += 1
        save_result(param_low, param_high, N, passes_completed,
                    total_eigdecomps, time.time() - t0)

        # If bracket converged, widen slightly for re-verification at larger N
        if param_high - param_low < 1e-6:
            mid = 0.5 * (param_low + param_high)
            param_low = mid - 0.005
            param_high = mid + 0.005

        # Deepen: double matrix size up to GPU memory limit
        N *= 2
        if N > MAX_N:
            N = MAX_N


if __name__ == "__main__":
    main()
```



### Time-Aware Loops (MANDATORY for ALL experiments)

Every experiment MUST use a time-aware main loop. NEVER use fixed iteration/epoch counts.

**Rules:**
- Use `deadline = time.time() + _run_duration * 0.85` (leave 15% for final output)
- Main loop: `while time.time() < deadline:` — do the next unit of work
- For bisection experiments: each iteration doubles N and refines the bracket
- For ANY other experiment type: each iteration does one more pass of computation (epoch, sweep, etc.)
- Record metrics periodically inside the loop AND once after the loop exits
- Fixed counts like `n_epochs = 120` or `N_EPOCHS = max(40, ...)` are BANNED — they waste time on fast machines and under-train on slow ones

**BAD (fixed count):**
```python
n_iterations = min(120, max(40, int(_run_duration // 6)))
for i in range(n_iterations):
    do_work()
```

**GOOD (time-aware):**
```python
deadline = time.time() + _run_duration * 0.85
while time.time() < deadline:
    do_work()
```


**CRITICAL RULES for GPU iterative deepening:**
- **ALL array operations use `xp`** (cupy on GPU, numpy fallback). NOT raw numpy.
- **Free GPU memory between passes:** `cp.get_default_memory_pool().free_all_blocks()`
- **Convert to numpy for JSON:** `cp.asnumpy(gpu_array)` before writing results.
- **Convert BACK to cupy after numpy.linalg:** If you call `numpy.linalg` on `cp.asnumpy(array)`, the result is a numpy array. You MUST convert it back to cupy (`xp.asarray(...)`) before any subsequent `xp`/cupy operations. Forgetting this causes silent CPU fallback or dtype errors.
- **Scale N from GPU FLOPS calibration.** No hardcoded sizes.
- **Cap N from VRAM.** `eigvalsh` needs ~3x the matrix memory for workspace.
- **ONE seed per workunit.** No multi-trial loops.
- **Time estimation is mandatory.** `if last_pass_time * 8 > remaining: break`
- **Write result after every completed pass.** The last written result IS the answer.
- **`_fitness` MUST be the FIRST key in `experiment_result`.** This is a single float that measures result quality — like Stockfish's depth+eval score. The science step uses it to sort results and focus on the most informative ones. For bisection experiments: `_fitness = 1.0 / max(bracket_width, 1e-15)`. For non-bisection experiments: define a meaningful quality metric (e.g., abs(effect_size), convergence_measure, trial_count). Higher = better.
- **Copy the boilerplate** (`build_ordered_experiment_output`, `load_context`, `NumpyEncoder`, etc.) from a CPU reference script. It works identically on GPU.

**RESOURCE LIMITS — VOLUNTEERS' MACHINES ARE NOT YOURS:**
- **Peak VRAM: stay under 80% of GPU_MEMORY_MB.** One matrix at a time, freed between passes.
- **Peak RAM: stay under 4GB.** Results and small numpy arrays only on CPU side.

- **Result JSON must stay under 1MB.** Output summary statistics only.

- **Example of BAD depth:** Grid that doubles every minute until GPU OOMs.

**NOTE — GPU PIPELINING (for experiments with multiple disorder samples per bisection step):**When running multiple eigvalsh calls in a loop (e.g., averaging over disorder samples), the GPU goes idle between calls while the CPU builds the next matrix. Volunteers see a sawtooth in nvidia-smi (100%% → 0%% → 100%%). Fix this by **pre-building the next matrix while the GPU is still crunching the current one** using CUDA streams:```pythondef test_with_pipeline(N, param, n_samples, rng):    total = 0.0    H_next = build_matrix(N, param, rng)    if _HAS_GPU:        stream_compute = cp.cuda.Stream(non_blocking=True)        stream_build = cp.cuda.Stream(non_blocking=True)    for i in range(n_samples):        H = H_next        if _HAS_GPU:            with stream_compute:                eigs = xp.linalg.eigvalsh(H)            del H            if i < n_samples - 1:                with stream_build:                    H_next = build_matrix(N, param, rng)            stream_compute.synchronize()            eigs_np = cp.asnumpy(eigs)        else:            eigs = xp.linalg.eigvalsh(H)            del H            eigs_np = eigs            if i < n_samples - 1:                H_next = build_matrix(N, param, rng)        del eigs        total += analyze(eigs_np)    return total / n_samples```The CPU queues the next matrix build (stream B) while the GPU is still running eigvalsh (stream A). The GPU always has work waiting — no idle gaps between eigvalsh calls. This is standard practice in scientific GPU computing. **Use this pattern whenever your inner loop runs multiple GPU operations per bisection step.**
