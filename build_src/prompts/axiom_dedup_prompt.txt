## Task

1. Determine today's date: `date -u '+%Y-%m-%d'`

**Optional — Science Journal (long-term memory):** For context from earlier analysis cycles today:
```bash
cat /opt/axiom/experiment_session/science_memory/science_memory_$(date -u +%Y-%m-%d).txt 2>/dev/null
```

2. Read ALL `results_*.txt` files in `/opt/axiom_boinc/html/user/experiment_results/` from today only (less than 1 day old).

3. Parse every `EXPERIMENT:` block from each file. Extract:
   - Experiment name (normalized: lowercase, strip whitespace, strip leading "exp_" prefix, strip trailing "_gpu" suffixes, remove "(GPU)" tags, replace hyphens and spaces with underscores, collapse multiple underscores)
   - Sample count (the N from "N results" or "N seeds")
   - Number of hosts
   - Results line
   - Conclusion line (and any indented explanation lines below it)
   - The report filename and its mtime

4. Group by normalized experiment name. For duplicates, keep ONLY the **best** entry — defined as:
   - Highest sample count (results or seeds, whichever is larger)
   - If tied, most recent report file

5. **Exclude "insufficient data" findings.** Remove any experiment whose Conclusion contains "insufficient data" (case-insensitive).

6. Write the deduplicated findings to today's daily file ONLY:
   `/opt/axiom_boinc/html/user/experiment_results/daily_dedup/dedup_YYYY-MM-DD.txt`

Format:
```
AXIOM DEDUPLICATED FINDINGS — auto-generated, do not edit
Generated: YYYY-MM-DD HH:MM UTC
Source: results for YYYY-MM-DD

EXPERIMENT: Name (N results, N seeds, N hosts)
  Discovered: YYYY-MM-DD HH:MM UTC
  Results: ...
  Conclusion: STATUS. 2-3 sentence explanation on the SAME line.
```

**CONCLUSION FORMAT IS CRITICAL:**
- Conclusion MUST be a SINGLE LINE: `Conclusion: STATUS. Explanation text here.`
- NEVER split across multiple lines. NEVER use indented continuation lines.
- NEVER write `Conclusion: : STATUS` (double colon) — only `Conclusion: STATUS.`
- NEVER write bare status without explanation (e.g., just `Conclusion: REJECTED`)
- NEVER put raw stats (d=, seeds=, fraction=) as the conclusion — those belong in Results
- If the source report has a bare conclusion (e.g., `Conclusion: CONFIRMED`), copy it as-is — do NOT add a colon or reformat it
- If the source has explanation text on a separate indented line, merge it onto the Conclusion line

**IMPORTANT format rules:**
- Copy the EXPERIMENT header line VERBATIM from the best source report. Do NOT reformat or add extra parenthesized groups.
- **Preserve experiment names exactly.** The website auto-detects STEM categories by matching keywords in experiment names. Do NOT rename, rephrase, or clean up names.
- The `Discovered:` line goes right after the EXPERIMENT header, BEFORE Results.
- Sort findings by discovery date (newest first).

7. After writing the daily file, rebuild the master by running:
   `python3 /opt/axiom/experiment_session/rebuild_master_from_daily.py`

8. Print a one-line summary: "Deduplicated: K entries -> M unique findings"
