### PREFERRED PATTERN: Iterative Deepening Bisection

**This is the PRIMARY pattern for all new CPU experiments.** Like Stockfish's chess search: start at small matrix size, bisect a parameter, deepen until time runs out. Always have a current best answer on disk.

#### The iterative deepening pattern:
1. Start at small problem size (N=64)
2. Do 8 binary search steps at that size (bisect the parameter)
3. Write result to disk (checkpoint — the answer is always ready)
4. Free the matrix (garbage collect — memory stays flat)
5. Carry forward ONLY the bracket (two floats: low, high)
6. Double N (64 → 128 → 256 → 512 → 1024 → 2048)
7. Bisect within the narrowed bracket from the previous pass
8. **TIME ESTIMATE: if `last_pass_time * 8 > time_remaining`, STOP** — don't start a pass you can't finish
9. Time runs out → last written result IS the answer

#### Why this works:
- **Matrix size IS the depth** (like search depth in chess). Not a fixed parameter.
- Heavy op is `np.linalg.eigvals()` or `eigvalsh()` — **O(N³)**. CPU has no choice but to grind.
- Each pass is **~8x heavier** than the last (N doubles, 2³=8). Early passes are free, last pass eats the budget.
- **No 99% hang**: time estimation prevents starting a pass that can't finish.
- **Memory stays flat**: one NxN matrix at a time, freed between passes. Peak ~32MB at N=2048.
- Slow CPU reaches pass 3, fast CPU reaches pass 6+. **Same code, hardware determines depth.**
- Aggregation happens at BOINC level: hundreds of seeds × different random realizations = statistical power.

#### What makes a GOOD iterative deepening experiment:
- A **parameter with an unknown critical threshold** to find (phase transition, stability boundary, universality breakdown)
- An **observable** that cleanly distinguishes two phases (fraction, ratio, density, level spacing ratio)
- A **matrix operation that scales as O(N³)** — `eigvalsh()`, `eigvals()`, SVD
- A **genuinely OPEN problem** — no known analytical solution exists
- The **bisection parameter** should have a roughly constant critical value across N. If it changes with N, bisect a **rescaled variable** instead (e.g., `alpha = b/sqrt(N)`, or `gamma` where `p = N^{-gamma}`)

#### Study these deployed reference experiments:
- `spiked_matrix_heavy_tail.py` — BBP transition with Student-t noise (beta_c unknown for heavy tails)
- `dale_law_stability.py` — Dale's law stability threshold vs May's prediction (open neuroscience problem)
- `band_matrix_wigner_poisson.py` — Anderson localization bandwidth scaling (alpha_c prefactor unknown)
- `sparse_wigner_universality.py` — Tracy-Widom universality breakdown at finite sparsity (gamma_c unknown)

Read one of these before writing your own. They show the complete pattern with boilerplate.

Heavy op: np.linalg.eigvalsh() on NxN symmetric matrix — O(N^3).

_seed, _seed_source_flag, _run_duration = load_context()

INITIAL_N = 64
DISORDER_SAMPLES = 5       # matrices per bisection test
BISECTION_STEPS = 8
PARAM_LOW_INIT = 0.1       # initial bracket — must contain the true threshold
PARAM_HIGH_INIT = 5.0
OBSERVABLE_THRESHOLD = 0.5  # criterion for bisection decision


def build_matrix(N, param, rng):
    """Build NxN matrix parameterized by the bisection variable.
    This is where your physics goes — the matrix encodes the problem.
    """
    # Example: symmetric random matrix with structure controlled by param
    H = rng.randn(N, N)
    H = (H + H.T) / 2.0  # symmetrize
    # ... apply your specific construction using param ...
    return H


def measure_observable(N, param, n_samples, rng):
    """Compute the observable that distinguishes the two phases.
    Returns a scalar — the bisection decides based on whether this
    is above or below OBSERVABLE_THRESHOLD.
    """
    total = 0.0
    for _ in range(n_samples):
        M = build_matrix(N, param, rng)
        eigs = np.linalg.eigvalsh(M)  # O(N^3) — the heavy operation
        # ... compute your criterion from the eigenvalues ...
        total += your_criterion(eigs)
    return total / n_samples


def save_result(p_low, p_high, current_N, passes_completed,
                total_eigdecomps, elapsed):
    """Write current best answer to disk (anytime algorithm)."""
    _bw = max(float(p_high - p_low), 1e-15)
    result = {
        "experiment_result": {
            "_fitness": float(1.0 / _bw),  # MUST BE FIRST KEY — tighter bracket = higher fitness
            "trials_completed": 1,
            "elapsed_sec": float(round(elapsed, 2)),
            "seed": int(_seed),
            "seed_source": float(_seed_source_flag),
            "run_duration_requested_sec": float(_run_duration),
            "summary": {
                "current_matrix_size": float(current_N),
                "param_low": float(p_low),
                "param_high": float(p_high),
                "param_c_estimate": float(0.5 * (p_low + p_high)),
                "bracket_width": float(p_high - p_low),
                "passes_completed": float(passes_completed),
                "total_eigdecomps": float(total_eigdecomps),
            },
            "hypothesis": {
                "param_c_estimate": float(0.5 * (p_low + p_high)),
                "bracket_width": float(p_high - p_low),
                "deepest_N": float(current_N),
                "passes_completed": float(passes_completed),
            },
        }
    }
    with open("experiment_result.json", "w") as fh:
        json.dump(build_ordered_experiment_output(result, globals()),
                  fh, indent=2, cls=NumpyEncoder)

    rng = np.random.RandomState(_seed)

    param_low = PARAM_LOW_INIT
    param_high = PARAM_HIGH_INIT

    total_eigdecomps = 0

            if time.time() + last_pass_time * 8 > deadline:

        samples = max(2, DISORDER_SAMPLES - passes_completed)

                    total_eigdecomps, time.time() - t0)

            mid = 0.5 * (param_low + param_high)
            param_low = mid - 0.005
            param_high = mid + 0.005

        # Deepen: double matrix size
        N *= 2
        if N > 2048:
            N = 2048
