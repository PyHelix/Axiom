# Axiom BOINC — Experiment Descriptions (Step 14 of 14)

You are the experiment descriptions agent for the Axiom BOINC project. Your ONLY job is to ensure every experiment has a human-readable description in the website descriptions JSON file. This makes experiments clickable on the experiments page, linking to detail pages that explain what each experiment does.

## Why This Step Exists

When the Research & Deploy steps (7-10) deploy new experiments, they create Python scripts with docstrings but no website description. This step reads those docstrings and writes human-friendly summaries so volunteers can understand what science their machines are running.

Descriptions persist forever — they stay in the JSON file even after experiments retire to Completed/Retired status. The website already checks descriptions for both Active and Completed tables.

## Step 1: Find Experiments Missing Descriptions

Read the existing descriptions file:
```bash
cat /opt/axiom_boinc/html/user/experiment_descriptions.json
```

Read the stats cache to get all known experiment names:
```bash
cat /opt/axiom_boinc/html/user/experiment_stats_cache.json
```

Extract all experiment names from both the `live` and `completed` arrays in the stats cache. Compare against the keys in the descriptions JSON. Identify any experiments that exist in the stats cache but NOT in the descriptions JSON.

If ALL experiments already have descriptions, print "All experiments have descriptions — nothing to do." and stop.

## Step 2: Read Script Docstrings

For each experiment missing a description, read its script:
```bash
head -60 /opt/axiom_boinc/html/user/experiments/EXPERIMENT_NAME.py
```

The script's docstring (triple-quoted string at the top) contains:
- A title/summary line
- An "Open problem" paragraph explaining the scientific question
- A "Novelty check" section (skip this — it's internal)
- Method details (iterative deepening, eigensolve, etc.)

If the script file doesn't exist, skip that experiment.

## Step 3: Write Description

For each missing experiment, create a description entry with these fields:
- **title**: Human-readable title (Title Case, derived from the script's first docstring line)
- **category**: One of: Physics, Ecology, Statistics, Network Sci., Pop. Genetics, Epidemiology, Nonlinear Dyn., Machine Learning, Number Theory, Neuroscience, Science (fallback)
- **summary**: One sentence summarizing the experiment's goal (from the open problem paragraph)
- **description**: 2-3 paragraphs explaining: (1) the scientific question, (2) how the experiment works, (3) why this hasn't been done before. Written for an educated non-specialist audience. Do NOT include the novelty check URLs or search terms.
- **method**: One sentence about the computational method (e.g., "Dense eigensolve with iterative deepening. System sizes N=64 to 2048.")
- **fields_measured**: What the experiment outputs (e.g., "Critical threshold, participation ratio, transition sharpness.")

## Step 4: Update the JSON File

Read the current JSON, add the new entries, and write it back:
```bash
python3 << 'PYEOF'
import json

# Read existing
with open("/opt/axiom_boinc/html/user/experiment_descriptions.json", "r") as f:
    descriptions = json.load(f)

# Add new entries
new_entries = {
    # ... your new entries here ...
}
descriptions.update(new_entries)

# Write back (sorted keys for consistency)
with open("/opt/axiom_boinc/html/user/experiment_descriptions.json", "w") as f:
    json.dump(descriptions, f, indent=2)

print(f"Added {len(new_entries)} new description(s)")
PYEOF
```
