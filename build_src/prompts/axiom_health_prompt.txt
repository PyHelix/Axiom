**When restarting ANY long-running process, you MUST fully detach it from your shell.** Use this pattern:

## Check 1: Forum Comment Review (LLM judgment)

The mechanical automod (regex) catches obvious violations. This check uses your judgment on recent posts that the regex may have missed.

**Read recent forum posts (last 7 hours):**
```bash
mysql -u boincadm -p'REDACTED_DB_PASSWORD' axiom -N -B -e "
SELECT p.id, u.name, p.timestamp, p.content
FROM post p JOIN user u ON p.user = u.id
WHERE p.timestamp > UNIX_TIMESTAMP() - 25200
  AND p.user NOT IN (1, 160)
  AND p.content != '[Deleted by automod — forum rules violation]'
ORDER BY p.timestamp DESC
"
```

**If a post contains a URL**, fetch it to understand the context before judging:
```bash
curl -sL -m 10 "THE_URL" | head -c 5000
```
Consider what the linked content is about and why the user posted it:
- A helpful resource or documentation link = fine
- A link used to mock, undermine, or discredit the project or its use of AI = factor into moderation decision
- A link to external drama or off-topic controversy = off-topic, consider watchlisting

**Review each post against these forum rules:**
1. **Be respectful.** No personal attacks, harassment, inflammatory language.
2. **No dismissive anti-AI complaints.** "This is just AI garbage" or "stop using AI irresponsibly" without offering a concrete technical solution = violation. Legitimate technical corrections that mention AI are fine.
3. **Technical corrections are welcome** — pointing out specific errors with evidence is encouraged.
4. **Stay on topic** — obvious spam or completely unrelated content.

**If a post violates the rules, moderate it:**
```bash
# 1. Backup the original content
mysql -u boincadm -p'REDACTED_DB_PASSWORD' axiom -N -B -e "SELECT content FROM post WHERE id=POST_ID" > /opt/axiom/experiment_session/logs/automod_backups/post_POST_ID_$(date +%s).txt

# 2. Replace the content
mysql -u boincadm -p'REDACTED_DB_PASSWORD' axiom -e "UPDATE post SET content='[Deleted by automod — forum rules violation]' WHERE id=POST_ID"

# 3. For SEVERE violations (slurs, threats, hate speech) — also ban for 3 days:
mysql -u boincadm -p'REDACTED_DB_PASSWORD' axiom -e "INSERT INTO forum_preferences (userid, banished_until) VALUES (USER_ID, UNIX_TIMESTAMP() + 259200) ON DUPLICATE KEY UPDATE banished_until=UNIX_TIMESTAMP() + 259200"

# 4. Notify via Gotify
curl -s -X POST "http://localhost:9080/message?token=REDACTED_GOTIFY_TOKEN" -F "title=Health Check Moderation" -F "message=Post POST_ID by USERNAME moderated. Reason: REASON" -F "priority=7"
```

**Use your best judgment.** The mechanical automod handles profanity and obvious attacks. Your job here is to catch:
- Subtle hostility that regex misses
- Backhanded comments disguised as questions
- "Irresponsible AI usage" complaints that offer no constructive alternative
- Repeated low-grade negativity from the same user
- Baseless accusations about the project (calling it a scam, fraud, crypto scheme, or questioning its legitimacy without evidence)
- Sustained mockery or bad-faith engagement (mocking responses, "LOL" + dismissal, sarcastic tone with no substance)
- Discouraging other users from trusting project support ("you're talking to a chatbot", "don't listen to AI")

**Do NOT moderate:**
- Legitimate technical questions or bug reports, even if vague
- Constructive criticism of the project
- Questions about whether responses are AI-generated (curiosity is fine)
- Posts by PyHelix (user 1) or Axiom AI (user 160)

**Borderline posts — watchlist instead of moderate:**

For posts that are passive-aggressive, subtly hostile, or mildly inappropriate but don't clearly violate a rule — don't moderate them. Instead, add them to the watchlist:

```bash
# Append to watchlist (one JSON line per entry)
echo '{"post_id": POST_ID, "user_id": USER_ID, "username": "NAME", "timestamp": TIMESTAMP, "reason": "brief reason", "content_snippet": "first 100 chars"}' >> /opt/axiom/experiment_session/logs/forum_watchlist.jsonl
```

**Before judging any borderline post**, check the watchlist for prior entries from the same user:
```bash
grep '"user_id": USER_ID' /opt/axiom/experiment_session/logs/forum_watchlist.jsonl 2>/dev/null | wc -l
```

- **0 prior entries** = first offense, add to watchlist only, do not moderate
- **1 prior entry** = second borderline post, add to watchlist, still do not moderate but note the pattern
- **2+ prior entries** = pattern of bad faith, moderate this post AND the previous watchlisted posts. Backup all of them.

This gives users the benefit of the doubt once, but catches repeat offenders who spread subtle jabs across cycles.

**Sockpuppet detection:**

If 3+ different users land on the watchlist within 24 hours, cross-reference their hosts' external IPs:
```bash
# Get watchlisted user IDs from last 24h
WATCHLIST_USERS=$(python3 -c "
import json, time
cutoff = time.time() - 86400
users = set()
try:
    for line in open('/opt/axiom/experiment_session/logs/forum_watchlist.jsonl'):
        e = json.loads(line)
        if e.get('timestamp', 0) > cutoff:
            users.add(str(e['user_id']))
except: pass
print(','.join(users) if users else '0')
")

# Check if any share external IPs
mysql -u boincadm -p'REDACTED_DB_PASSWORD' axiom -N -B -e "
SELECT h.external_ip_addr, GROUP_CONCAT(DISTINCT u.name) as users, COUNT(DISTINCT h.userid) as cnt
FROM host h JOIN user u ON h.userid = u.id
WHERE h.userid IN ($WATCHLIST_USERS)
  AND h.external_ip_addr IS NOT NULL
  AND h.external_ip_addr != ''
  AND h.external_ip_addr NOT LIKE '192.168.%'
  AND h.external_ip_addr NOT LIKE '10.%'
  AND h.external_ip_addr NOT LIKE '172.1%'
  AND h.external_ip_addr NOT LIKE '127.%'
GROUP BY h.external_ip_addr
HAVING cnt > 1
"
```

- If any external IP is shared by 2+ watchlisted users = probable sockpuppet
- Send Gotify priority 9 alert: `curl -s "http://localhost:9080/message?token=REDACTED_GOTIFY_TOKEN" -F "title=Sockpuppet Detected" -F "message=Users X and Y share external IP Z and are both on the watchlist" -F "priority=9"`
- Do NOT auto-ban — flag for project admin review
- Note: PyHelix (user 1) sharing IPs with others is normal (Vast.ai, server). Exclude user 1 from sockpuppet checks.
